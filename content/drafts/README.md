# content/drafts

Working drafts for the site's content strategy. **Not published.**

Gatsby only sources `content/blog` and `content/assets` (see `gatsby-config.js`), so nothing in this folder is built into a page. These files are tracked in git for versioning and collaboration, but they never reach the live site until moved into `content/blog` (or a dedicated collection) on purpose.

## Layout

- `<slug>/plan.md` : one plan per article. Working method is plan-only: title (EN + FR), a bulleted outline mirroring the article sections, and LinkedIn post angles (angles only, no post text yet). Grounded in real code / real research; unresolved facts are marked `[à confirmer: ...]`.
- `hall-of-fame/candidates.md` : candidate entries for the "AI Hallucinations Hall of Fame", bilingual clickbait titles, each grounded and anonymized. Not for publication without confirmation and screenshots.
- `_archive-prose/<slug>/` : earlier full-prose drafts (en / fr / linkedin) kept for reference. Superseded by the plan-only method.

## Rules carried in these drafts

- Anonymized: no client or product names.
- No em-dashes anywhere.
- No fabricated metrics: unknowns are `[à confirmer]`.
- Security / research pieces stay defensive: no exploit payloads.
